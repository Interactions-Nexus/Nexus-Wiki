低层库 (LLL)

低层库 (LLL) 是 TAO (麟道）框架的基础。它是一个用 C++ 编写的基本模板库，可最大限度地减少匹配硬件，且最大限度地提高裸机效率。在架构上，LLL 低层是一种可互换的结构，需要为特定功能逻辑开发的模板和模块。 LLL-TAO 或 TAO 框架是一系列 LLL 低层库模板和数据模型，可通过简单的基于 JSON 的 API 應用程式介面访问，允许任何类型的开发人员提高其应用程序的安全性、可扩展性和稳固性。

LLL 低层库包含三个主要组件：低层加密 (LLC)、低层数据库 (LLD) 和低层协议 (LLP)。堆栈中有几种 LLD 低层数据库表示形式；分类帐本、注册、操作和 API 應用程式介面。 LLC 低层加密主要应用于分类帐层面，尽管可以应用在其他地方。作为网络层的一个组件，LLP 低层协议被设计为一种轻便、快速的协议，允许开发人员自定义他们的数据包设计和信息注解。

 

Lower Level Crypto LLC 低层加密

处理加密的一组操作流程，包括：

  • 数字签名（ECDSA，基于哈希）
  • 哈希函数（SHA3 / 著名的安全算法）
  • 加密（对称/非对称）
  • 后量子密码法（实验）

 

目前已经实施的低层加密 LLC：

  • SK 哈希（Skein 和 Keccak）
  • Argon2 密码哈希
  • AES 标准（对称）
  • FALCON（抗量子点阵签名）
  • OpenSSL 包装函式（EC_KEY、BIGNUM）

 

Lower Level Database LLD 低层数据库

用于设计高效数据库系统的模板集。核心模板可以扩展到更高级别的数据库类型。

  • 钥匙链数据库
  • ACID 交易
  • 扇区数据库

 

钥匙链包括：

  • 二进制文件映射
  • 二进制哈希映射

我们欢迎任何新钥匙链的贡献，以提供扇区数据的不同索引数据结构。

 

Lower Level Protocol LLP 低层协议

用于高效处理客户端/服务器数据的模板集。继承和创建自定义数据包类型，以轻松编写新协议，无需网络编程。

  • 数据服务器
  • 监听服务器
  • 连接类型
  • 数据包裹样式
  • 事件触发器
  • DDOS 限制器

 

目前已经实施的 LLP 低层协议：

  • Legacy 传统
  • Tritium 麟氚
  • HTTP 超文本传输协议
