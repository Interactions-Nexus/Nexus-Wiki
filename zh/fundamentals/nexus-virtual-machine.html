Nexus (NVM) 麟社虚拟机

麟社虚拟机 (NVM) 是一种状态机，作为由数百个链接的麟社节点维护的单个实体存在。

麟社软件堆栈的存在作用是为了维护这个特殊状态机的持续工作、不间断和不可逆变的操作；它是所有麟社帐户和增强合同都存在的环境。在区块链中的任何定点，麟社都维持着单一的规范状态，而 NVM 麟社虚拟机定义了从一个区块到生成另一个区块计算新的有效状态的规则。

尽管所有智能合约平台都是状态机，但在现实世界中必须具备可扩展性，最终造就了 NVM 麟社虚拟机的独一无二的架构。为了轻松区分更好的架构，必须将麟社上的合约命名为增强合约。下面我们将解开架构。

 

NVM 麟社虚拟机架构

NVM 麟社虚拟机被设计为 “64 位元”、“基于寄存器”；这种设计选择是因为它匹配了 CPU 的 64 位元并模仿 CPU 缓存寄存器。

与 EVM 以太坊虚拟机相比之下，NVM 麟社虚拟机这种设计非常快，它是专为当今的处理器而设计。用数字来表示，EVM 需要 170 万纳秒/指令，NVM 则只需要 33 纳秒/指令。这是由于在 64 位元 CPU 上的 256 位元长度以及过时的堆栈设计，EVM 处于一个巨大的劣势，因为它需要 4 个周期才能完成一条指令。

NVM 麟社虚拟机特意被设计为不是图灵完备性。这一决定还源于麟社是一个验证引擎这一事实。这种设计有一个巨大的好处，那就是免费的简单交易，而 EVM 以太坊虚拟机需要 Gas 来控制由于代码错误而导致的计算请求，这可能会使网络陷入瘫痪。使用 NVM 麟社虚拟机设计，增强合同将具有可预测的费用，这些费用将在合同执行之前就计算出来。

麟社将有不同类型的合约，对于更高级别的 API，将提供模板，用户可以从列表中拉下选择。对于进阶用户，增强合同将使他们能够使用他们选择的领域特定语言的合同。增强合同将在以后提供。详细的资料，请参阅麟社开发路线图。
