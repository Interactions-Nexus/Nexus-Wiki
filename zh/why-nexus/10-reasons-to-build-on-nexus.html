1: Platform-agnostic 与平台无关的 API

Nexus 麟社在设计之初就考虑到了可组合性。 团队和社群一直在构建一个 “互联世界”，只有在主流 Web2 基础设施（如市场、开发平台、自动化工具、游戏引擎和社交网络）与新兴的 Web3 之间建立一个去中心化的桥梁才能实现， 通过 Nexus 麟社的集成 API（应用程序编程接口）工具包。 基于 Nexus 麟社构建的去中心化应用程序 (DApps) 与 JSON-API 交互，允许开发人员将区块链功能集成到他们的传统应用程序中，同时具有基于 Web 开发的技能。 Nexus 麟社提供了广泛的经过良好测试的命令，可以减少 Dapp 开发中涉及的时间、风险和成本。 Nexus 麟社独特的架构支持数据丰富的 Web2 和通证化 Web3 之间的无缝互操作性。

 

2: Fueling the No-Code Movement 推动无程式码运动

Bubble 是无程式码运动的领导者，它提供了一个强大的点击式 Web 编辑器和云托管平台，允许用户构建完全可定制的 Web 应用程序和工作流。 Nexus 的 Bubble 插件包含几个直接与 Nexus 麟社区块链交互的 API 插件调用。 对于渴望为 Web3 时代构建通证化的点对点 NFT（非同质化通证）市场、视觉艺术画廊、音乐和视频流平台、社交网络以及许多其他服务、创作和想法的人来说，它是一个强大的工具 .

3: Easy Token & Asset (NFT) Creation 轻松创建通证和资产（NFT）

可以通过 Nexus 麟社钱包或 Nexus API 麟社插件轻松创建通证和资产 (NFT)。 创建通证的费用取决于总供应量，而创建 NFT 的费用为 1 NXS 麟社币。 与常规通证不同，NFT 是区块链上的独特资产，可用于代表许多不同类型的数字和实物资产，例如数字艺术、实物收藏品和音乐。 NFT 也可以被通证化，允许共享持有权和自动分配版税或股息。 在这里查看我们的费用表。

4: Namespaces 名称空间

Nexus has a multi-tiered naming system that offers globally unique identification, personalization and branding opportunities. Three categories exist: local names, namespaces, and global names, all of which can be created via the Nexus desktop wallet. A user only has to type in the username and account name to send funds to another user (eg. send to jules:default). Names can be created within a namespace for the purpose of naming accounts within an organization. A namespace owner can create and sell names just like domain registrars sell domains. Global names represent a unique, stand-alone name that can be used for token tickers and other uses. Global name ownership can be transferred just like namespaces, creating the opportunity for secondary markets.

Nexus 拥有多层命名系统，可提供全球唯一的标识、个性化和品牌推广机会。 存在三个类别：本地名称、命名空间和全局名称，所有这些都可以通过 Nexus 桌面钱包创建。 用户只需输入用户名和账户名即可将资金发送给另一个用户（例如发送至 jules:default）。 名称可以在名称空间内创建，用于命名组织内的帐户。 命名空间所有者可以创建和出售名称，就像域名注册商出售域名一样。 全局名称表示可用于代币行情和其他用途的唯一、独立的名称。 全球名称所有权可以像命名空间一样转让，为二级市场创造机会。

5: Mobile-friendly Automated Key Management

Nexus has built a groundbreaking transactional architecture called ‘Signature Chains’ that replaces the legacy UTxO, and so-called account based models, to simplify application development, wallet usability, and therefore boost user adoption. A Signature Chain is a personal vault containing the historical record of every single transaction and digital asset ownership of a single user on the blockchain, and is accessed with user-generated credentials (username, password, and PIN) by a decentralized login system. Signature Chains make Nexus the perfect mobile-friendly Web3 development stack by eliminating manual dependency on key management plugins such as MetaMask, enabling developers to build tokenized Dapps with simplified UI/UX that are as easy to use as Facebook, Snapchat, and TikTok.
 

6: Quantum-Proof Security

Signature Chains also enhance the security of existing ECDSA (Elliptic Curve Digital Signature Algorithm). They generate one-time-use key-pairs derived from the Nexus wallet credentials, of which the public key’s hash is only published once the key-pair is used, while deterministically generating a new key-pair while signing the next transaction. This results in a higher level of security compared to other platforms as the attack window to brute force a private key is reduced to under 200 milliseconds. In addition, Signature Chains utilize the following cryptographic functions: FALCON (a second round contender for the NIST Post-Quantum cryptography competition), Argon2 (winner of the password hashing competition, and a superior alternative to S-Crypt or B-Crypt), and Keccak (winner of the SHA3 competition) to improve its robustness.

 

7: Feeless Transactions

It is free to send and receive NXS, tokens and assets (NFTs). However, there is a 0.01 NXS fee applied to every simple contract generated within 10 seconds of the last transaction to prevent transaction spam attacks on the network. Each transaction on Nexus executes a contract and each transactional-contract can contain a maximum of 100 additional contracts. View our fee schedule here.

 

8: High Throughput

Our latest test shows that the current release can achieve 655 transactions per second on a live application, with each transaction able to contain up to 100 contracts.
 

9: Custom Database

Nexus has developed its own database called the Lower Level Database (LLD). It is a fast and modular storage engine that is capable of outperforming most of the current embedded database engines that exist. Tests have shown an average of 0.33 seconds per 100,000 writes and reads to disk, making it faster than Google’s LevelDB. LLD multithreading demoing from May 2021 showed 1024 threads at 578M reads per second, and 3333 threads reaching 4.2B reads per second. It is anticipated that staying above 1M reads per second will be the standard during live application.
