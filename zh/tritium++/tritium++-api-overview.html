Nexus 麟社 API 概览

Nexus 麟社提供了多个 API 应用程式界面以允许用户和开发人员轻松地与 Nexus 麟社软件堆栈的功能进行交互。每个 API 应用程式界面都是按功能区域与堆栈的特定部分（例如分类帐本、寄存器）交互的逻辑分组方法集合（账户、资产、通证），或按行业特定用例（供应、音乐）。 

 

Protocol 协议

API 服务器使用 HTTP 协议并同时支持 GET 和POST 请求方法。

API 使用 JSON (JavaScript Object Notation) 作为输入和输出的主要数据传输机制。但是 API 服务器也支持 URL 编码请求 (application/x-www-form-urlencoded)。

 

Port 端口

在默认情况下，API 服务器在端口 8080 上运行。这可以通过在 nexus.conf 中设置 apiport=xxxx 或使用 -apiport=xxxx 参数启动守护程序来更改。

 

Security 安全

Nexus 麟社 API 使用 HTTP Basic 身份验证方案。 这要求调用者设置一个授权 Authorization HTTP 标头，其值为 Basic <credentials> ，其中 <credendtials> 部分设置为 apiuser:apipassword 和base64 编码。 在 API 服务器端，通过将 apiuser=<username> 和 apipassword=<password> 添加到 nexus.conf 来配置身份验证。

注意：如果您希望完全禁用身份验证，可以通过在 nexus.conf 中设置 apiauth=0 来实现

身份验证机制由 SSL 加密 (HTTPS) 的实施补充。 默认情况下，API 服务器在禁用 SSL 的情况下启动。 要启用它，请将 apissl=1 添加到您的 nexus.conf

 

Multi-user Sessions 多用户会议

Nexus 麟社 API 可以配置为在多用户模式下运行，允许多个签名链同时登录并使用该 API。 可以通过在 nexus.conf 中设置 multiuser=1（默认关闭）来启用此功能。 启用多用户后，初始登录回复将包含一个会话 ID 值。 然后，此会议 ID 必须包含在对端点的所有后续 API 调用中，这些端点需要用户登录。

 

Endpoint 端点

可以通过以下语义调用每个 API 端点：

<api>/<verb>/<noun>

例如：

http://127.0.0.1:8080/ledger/get/block

注意：URI 总是小写的。

 

Error Handling

In the event that an API call results in an error, a JSON object will be returned with the following format:

{
    "error" :
    {
        "code" : -123,
        "message" : "An error has occurred"
    }
}
