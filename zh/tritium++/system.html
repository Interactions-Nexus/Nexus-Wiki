SYSTEM 系统

系统 API 提供对此节点信息的公共访问。 这包括节点运行的软件版本、分类帐本和内存池状态、节点 IP 地址以及连接的对等点数量等数据。

Direct Endpoints 直接端点

以下命令是直接端点，因此不支持上面可用的上述动词和名词结构。

stop
get/info
get/metrics
list/peers
list/lisp-eids
validate/address

直接端点支持过滤器和运算符。

stop 停止

启动节点关闭序列。

system/stop

Parameters 参数

password 登录密码：在启动核心关闭之前可选择进行 authenticate 身份验证。 此密码是配置文件中提供的密码，有助于防止意外远程关机。

注意：关机验证密码必须与标志一起提供 -system/stop=<password>

Results 结果：

返回值 JSON 物件：

"Nexus server stopping"
[Completed in 0.039270 ms]

返回值

version 版本：守护程序软件版本。

protocolversion 协议版本：低层协议 (LLP) 版本。

walletversion 钱包版本：传统链钱包版本。

timestamp 时间戳：此节点报告的当前统一时间。

hostname 主机名：本地计算机主机名。

ipaddress ip地址：本地计算机的公共 IP 地址，如您所连接的对等点所见。 在您至少有一个对等连接之前，这将是空白的。

testnet 测试网：如果此节点在测试网上运行，则显示测试网编号。

private 私有：该节点是否以私有模式运行。

multiuser 多用户：此节点是否在多用户模式下运行。

sessions 会议：仅在多用户模式下返回。 这显示当前登录到节点的用户会议数。

clientmode 客户端模式：指示此节点是否在客户端模式下运行的标志。

nolegacy 非传统链: 指示此节点是否已编译为支持旧版 UTxO 钱包的标志。 如果没有，则所有旧版钱包功能（通过旧版 RPC 可用）都不可用。 如果此标志是使用传统链支持编译的，则当前会从响应中省略它，因此只有在它为真时才会包含它。

blocks 区块：该节点的当前区块高度。

synchronizing 同步：指示此节点当前是否正在同步的标志。 不包括如果 private=true.

synccomplete 完整同步: 下载的块总数的百分比。

syncprogress 同步进度：当前同步完成相对于节点启动时间的百分比。

txtotal 交易数量：节点内存池中的交易数量。

connections 连接：对等连接数。

eids 端点标识符：此节点的 LISP 端点标识符 (EID) 数组。 除非此节点使用 LISP，否则不包括在内。

list/lisp-eids

This will return the LISP Endpoint Identifiers (EID's) currently configured for this node. If the lispers.net API is not running / available then this will return an empty array.

system/list/lisp-eids

Parameters:

-none-

validate/address

Validates a register / legacy address.

system/validate/address

Parameters:

address : Required to validate the base58 encoded register or legacy address to check.

Results:

Return value JSON object:

{
    "address": "8BuhE1y5oard3cEWqCSYK35bC6LdPRvcPR8N7GXYNatTw6De8eK",
    "is_valid": true,
    "type": "OBJECT",
    "object_type": "ACCOUNT"
}

Return values:

address : The address that was validated.

is_valid : Boolean flag indicating if the address is valid or not. For legacy addresses this indicates that the address if simply formatted correctly. For register addresses this flag checks that a register exists in the blockchain at the specified address.

type : For register addresses, the type of register at the address. Values can be APPEND, LEGACY, OBJECT, RAW, READONLY, RESERVED, SYSTEM or UNKNOWN.

object_type : If the type is OBJECT this field contains the standard object type of the object register. Values can be ACCOUNT, CRYPTO, NAME, NAMESPACE, REGISTER, TOKEN, TRUST or UNKNOWN.

is_mine : If the type is LEGACY this boolean flag indicates if the private key for the address is held in the local wallet.
