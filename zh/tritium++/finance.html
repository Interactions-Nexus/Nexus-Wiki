FINANCE 财务

 财务 Finance API 提供了在用户/账户之间发送和接收 NXS 麟社币或其他通证、创建账户和管理质押的方法。 财务 URI 的完整支持端点如下：

finance/verb/noun/filter/operator

URI 的最低要求组件是：

finance/verb/noun

 

受支持的动词

此 API 命令集当前支持以下动词：

create 创建 - 生成支持类型的新物件。
debit 借记 - 从支持的类型发行资金。
credit 贷记 - 索取从借记账户发放的资金。
burn 烧毁 - 从流通中删除给定的通证。
get 获取 - 获取支持类型的物件。
list 列出 - 列出给定用户拥有的所有物件。
history 历史 - 生成所有最后状态的历史。
transactions 交易 - 列出修改指定物件的所有交易。

受支持的名词

此 API 命令集支持以下名词：

[account] - 包含通证 ID 和余额的物件寄存器。
[trust] - 包含通证 ID、余额和信誉的物件寄存器。
[token] - 包含通证 ID、余额、供应量和小数的物件寄存器。
[any] - 一个物件选择名词，允许不同通证的混合账户。
[all] - 物件选择名词，用于收集给定通证类型的所有账户。

Direct Endpoints 直接端点

以下命令是直接端点，因此不支持上面可用的上述 verb动词和 noun 名词结构。

get/balances
get/stakeinfo
set/stake
void/transaction
migrate/accounts

直接端点支持过滤器和运算符。

create 创建

创建一个由给定名词指定的新物件寄存器。

finance/create/noun

此命令不支持 any或 all名词。

Parameters 参数：

pin 操作密码：锁定时需要。 此配置文件的 PIN 操作密码。

session 会议：参数 argument -multiuser=1 要求提供以标识用户会话。 对于单用户 API 模式，不应提供会话。

name : 作为 UTF-8 编码字符串的 noun name名词名称可选，它将生成指向新物件的名称物件寄存器。 如果名词是 token 通证，这将被创建为全局名称。

data 数据：可选用于任何名词，允许调用者向账户寄存器添加任意数据。

create/token 创建/通证

supply 供应：设置最大令牌供应量的 noun token 名词通证需要。

decimals 小数点: 设置有效小数点总数的 noun token名词通证需要。 默认为 2。

create/account 创建/账户

token 通证：noun account名词账户需要作为 Base58 编码地址或代码名称。 默认为 NXS。

debit 借记

扣除一定数量的名词指定的 NXS 麟社币或通证，将其发送到另一个账户或传统链 UTXO 地址。 只有 NXS 麟社币可以发送到旧地址。

finance/debit/noun

此命令支持任何通配符名词。

debit/account 借记/账户

这将从通证账户或通证地址中扣除一定数量的 NXS 麟社币或通证，以发送到接收账户。

debit/token 贷记/通证

这将从通证地址中扣除一定数量的通证以发送到通证帐户

 

Parameters 参数：

pin 操作密码：锁定时需要。 此配置文件的 PIN。

session 会议：参数 argument -multiuser=1 要求提供以标识用户会议。 对于单用户 API 模式，不应提供会议。

from 从：需要 identify 识别要借记的账户，这可以是名称或注册地址。 该名称的格式应为 name（对于全局通证名称）、username:name（对于本地名称）或 namespace::name（对于名称空间中的名称）。

amount 数额：要借记的 NXS 麟社币或通证的数量。

to 到：需要 identify 识别收款账户。 这可以是账户名称或注册地址。 该名称的格式应为 username:name（对于本地名称）或 namespace::name（对于名称空间中的名称）。 如果从签名链账户发送到传统链地址，address_to 还可以包含传统链 UTXO 地址。

reference 参考：可选字段允许调用者提供参考，然后接收者可以使用该参考将交易与订单号、发票号等相关联。参考是 0 到 18446744073709551615 范围内的 64 位无符号整数。

expires 期限：可选字段允许调用者指定借记交易的 expiration 到期时间。到期值是从交易创建时间开始的秒数，在该秒数之后交易不能再被接收者贷记。相反，当您对交易应用到期时，在过期时间之前你不能取消交易。如果 expires 设置为 0，交易将永远不会过期，使发送者永远无法取消交易。如果省略，默认的过期时间为 7 天（604800 秒）应用。

burn烧毁

此方法可用于从当前供应中永久取出通证，这一过程通常称为烧毁。 该方法将从通证账户中扣除并将通证发送回通证地址。 然而，交易包含一个条件，该条件将始终评估为 false，保证通证发行者或发送者不能贷记借方。 结果是烧毁的金额将始终出现在通证的 “待处理” 余额中。

该命令只支持 account账户名词。

Parameters 参数：

pin 操作密码：locked 锁定时需要。 授权交易的 PIN。

session 会议：参数 argument -multiuser=1要求提供以标识用户会议。 对于单用户 API 模式，不应提供会议。

name 名称：需要 identify  识别账户名称以从中扣除通证以进行烧毁。 该名称的格式应为 username:name（对于本地名称）或 namespace::name（对于名称空间中的名称）。 如果提供了 address 地址，则是可选的。

address 地址：需要 identify  识别账户的注册地址以从被烧毁的通证中扣除。 如果提供了 name 名称，则这是可选的。

amount 数额：需要烧毁的通证 amount  数量。

reference 参考：可选字段允许调用者提供 reference 参考，然后接收者可以使用该参考将交易与订单号、发票号等相关联。参考是 0 到 18446744073709551615 范围内的 64 位无符号整数。

list

This will list off all of the object register details specified by the noun.

This command supports the account, trust and token nouns.

list/account

This lists all the NXS and token accounts, except the trust account for the logged in user.

list/trust

This lists the trust account for the logged in user.

list/token

This lists all the token addresses for the logged in user.

Parameters:

session : Required by argument -multiuser=1 to be supplied to identify the user session. For single-user API mode the session should not be supplied.

where : An array of clauses to filter the JSON results. More information on filtering the results from /list/xxx API methods can be found at Queries.

set/stake

Creates a stake change request for a profile's trust account. This request will add or remove stake to set the stake value to the requested amount. If the new value is more than the current stake amount, it adds stake from the account balance. If the new value is less, it removes stake to the account balance (with appropriate trust penalty, if applicable).

Requests are saved locally and take effect with the next stake block found by staking the profiles's trust account. Because they are saved locally, you must continue to stake on the machine where it was created until the next stake block is found, or the request will not be processed. Until implemented, you can update the request by calling set/stake again.

To remove a stake change request, you can either set an expiration time, or set the amount equal to the current trust account stake.

Endpoint:

finance/set/stake

expires : Optional field to assign the number of seconds until the stake change request expires. A value of zero indicates it does not expire. Default is zero if not passed.

migrate/accounts

This method will migrate your legacy accounts to signature chain accounts, sending the balance across in the process. A new account will be created in your signature chain for each legacy account, with a corresponding matching name (unless flagged not to create names). The balance of each legacy account is sent to the newly created signature chain account in individual transactions. As such, each transaction incurs the default legacy fee of 0.01 NXS, which is deducted from the amount being migrated.

The method uses the arbitrary data field in the account object register to track which legacy account it was created from. As a result, it is possible to invoke this method multiple times, and each time it will sweep any NXS from legacy accounts to existing signature chain accounts (as well as creating any necessary new accounts).
