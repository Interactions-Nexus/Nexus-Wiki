控制台

麟社钱包的控制台区域是最 “技术” 的部分。 如果您对 CLI（命令行界面）感到满意，我们建议您使用它。

在控制台中，可以发出各种指令来向守护程序发送指令。 这对于调试您的钱包非常有用，并将用于开发更多功能，因为开发人员可以从我们区块链上的可用数据中推断出新模块。

在控制台中输入文本 ‘Help’ 帮助，然后按执行将提供所有控制台命令的完整列表，如下所示：

  • addmultisigaddress <["key","key"]> [account] - 向钱包添加一个需要签名的多重签名地址，每个密钥是一个联系地址或十六进制编码的公钥。 如果指定了 [account]，则将地址分配给 [account]。
  • backupwallet <destination> - 安全地将 wallet.dat 复制到目的地，目的地可以是目录或带文件名的路径。
  • checkwallet - 检查钱包的完整性。
  • dumpprivkey <NexusAddress> - 显示与 <NexusAddress> 对应的私钥。
  • echo [param]...[param] - 回显调用中提供的参数的测试函数。
  • encryptwallet <passphrase> - 使用 <passphrase> 密码词组加密钱包。
  • getaccount <Nexusaddress> - 返回与给定地址关联的帐户。
  • getaccountaddress <account> - 返回用于接收此帐户付款的当前 Nexus 麟社地址。
  • getaddressesbyaccount <account> - 返回给定帐户的地址列表。
  • getbalance [account] [minconf=1] - 如果未指定 [account]，则返回服务器的总可用余额。 如果指定了 [account]，则返回帐户中的余额。
  • getblock <hash> [txinfo] - txinfo 可选，打印更详细的交易信息。 返回具有给定块哈希的块的详细信息。
  • getblockcount - 返回最长区块链中的区块数。
  • getblockhash <index> - 返回 <index> 处最佳区块链中区块的哈希值。
  • getblocknumber - 已弃用。 使用 getblockcount.
  • getconnectioncount - 返回到其他节点的连接数。
  • getdifficulty - 将难度作为最小难度的倍数返回。
  • getinfo - 返回包含各种状态信息的物件。
  • getmininginfo - 返回包含挖矿相关信息的物件。
  • getmoneysupply <timestamp> - 返回由矿工、持有者、开发人员和大使生产的 Nexus 麟社币的总供应量。 默认时间戳是当前的统一时间戳。 时间戳记录为 UNIX 时间戳。
  • getnetworkhashps - 获取哈希通道的网络哈希率。
  • getnetworkpps - 获取每秒搜索的网络素数。
  • getnetworktrustkeys - 列出网络上的所有 Trust 信誉密钥
  • getnewaddress [account] - 返回用于接收付款的新 Nexus 麟社地址。 如果指定了 [account]（推荐），它会被添加到地址簿中，因此使用该地址收到的付款将记入 [account]。
  • getpeerinfo - 返回有关每个连接的网络节点的数据。
  • getrawtransaction <txid> - 返回一个 std::string，它是 <txid> 的序列化十六进制编码数据。
  • getreceivedbyaccount <account> [minconf=1] - 返回具有至少 [minconf] 确认的交易中具有 <account> 的地址收到的总数额。
  • getreceivedbyaddress <Nexusaddress> [minconf=1] - 返回 <Nexusaddress> 在至少 [minconf] 确认的交易中收到的总金额。
  • getsupplyrates - 返回包含设定时间间隔内的当前 Nexus 麟社币生产率的物件。 时间频率以 13 个月 28 天为基数，共计 364 天。 这是为了防止格里高利数字的错误。
  • gettransaction <txid> - 获取有关 <txid> 的详细信息。
  • help [command] - 列出命令，或获取命令的帮助。
  • importprivkey <PrivateKey> [label] - 将私钥（由 dumpprivkey 返回）添加到您的麟社钱包。
  • isorphan <hash> - 返回块是否为孤立区块。
  • keypoolrefill - 填充密钥池，需要设置钱包密码词组。
  • listaccounts - 返回以账户名作为密钥、账户余额作为值的物件。
  • listaddresses [max=100] - 返回地址列表。
  • listreceivedbyaccount [minconf=1] [includeempty=false] - [minconf] 是包括付款前的最少确认次数。 [includeempty] 是否包含尚未收到任何付款的帐户。 这将返回包含以下内容的物件数组：
    • "account" : 接收地址的帐户
    • "amount" : 该帐户的地址收到的总数额
    • "confirmations" : 最近交易的确认数值
  • listreceivedbyaddress [minconf=1] [includeempty=false] - [minconf] 是包括付款前的最少确认次数。 [includeempty] 是否包括没有收到任何付款的地址。 这将返回包含以下内容的物件数组：
    • "address" : 接收地址
    • "account" : 接收地址的账户
    • "amount" : 地址收到的总数额
    • "confirmations" : 最近交易的确认数值
  • listsinceblock [blockhash] [target-confirmations] - Get all transactions in blocks since block [blockhash], or all transactions if omitted
  • listtransactions [account] [count=10] [from=0] - Returns up to [count] most recent transactions skipping the first [from] transactions for account [account].
  • listtrustkeys - List all the Trust Keys this Node owns.
  • listunspent [minconf=1] [maxconf=9999999] ["address",...] - Returns array of unspent transaction outputswith between minconf and maxconf (inclusive) confirmations. Optionally filtered to only include txouts paid to specified addresses. The results are an array of objects, each of which has:
  • makekeypair [prefix] - Make a public/private key pair. [prefix] is optional preferred prefix for the public key.
  • move <fromaccount> <toaccount> <amount> [minconf=1] [comment] - Move from one account in your Wallet to another.
  • repairwallet - Repair wallet if checkwallet reports any problem.
  • rescan - Rescans the database for relevant wallet transactions.
  • reset - Restart all node connections
  • sendfrom <fromaccount> <toNexusaddress> <amount> [minconf=1] [comment] [comment-to] - <amount> is a real and is rounded to the nearest 0.000001 requires wallet passphrase to be set with walletpassphrase first.
  • sendmany <fromaccount> {address:amount,...} [minconf=1] [comment] - amounts are double-precision floating point numbers requires wallet passphrase to be set with walletpassphrase first.
  • sendrawtransaction <hex std::string> [checkinputs=0] - Submits raw transaction (serialized, hex-encoded) to local node and network. If checkinputs is non-zero, checks the validity of the inputs of the transaction before sending it.
  • sendtoaddress <Nexusaddress> <amount> [comment] [comment-to] - <amount> is a real and is rounded to the nearest 0.000001 requires wallet passphrase to be set with walletpassphrase first.
  • setaccount <Nexusaddress> <account> - Sets the account associated with the given address.
  • signmessage <Nexusaddress> <message> - Sign a message with the private key of an address.
  • stop - Stop Nexus server.
  • unspentbalance ["address",...] - Returns the total amount of unspent Nexus for given address. This is a more accurate command than Get Balance.
  • validateaddress <Nexusaddress>- Return information about <Nexusaddress>.
  • verifymessage <Nexusaddress> <signature> <message>- Verify a signed message.
  • walletlock - Removes the wallet encryption key from memory, locking the wallet. After calling this method, you will need to call walletpassphrase again before being able to call any methods which require the wallet to be unlocked.
  • walletpassphrase <passphrase> [timeout] [mintonly] - Stores the wallet decryption key in memory for [timeout] seconds. mintonly is optional true/false allowing only block minting. timeout is ignored if mintonly is true / 1.
  • walletpassphrasechange <oldpassphrase> <newpassphrase>- Changes the wallet passphrase from <oldpassphrase> to <newpassphrase>.
