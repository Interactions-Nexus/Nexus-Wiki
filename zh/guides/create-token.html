创建通证 (代币）

本指南将帮助用户使用 Nexus 麟社界面创建通证

在我们开始创建通证之前，用户需要熟悉通证使用的一些概念和参数。

Token Parameters 通证参数

通证参数是定义通证的参数，例如通证名称、供应量和小数位数。 在下面了解有关它们的更多信息：

TOKEN NAME 通证名称

这是标识通证的本地名称，是可选的。 此名称位于用户帐户 (Sigchain) 签名链内，在其外部无效。

通证名称为 1NXS 麟社币的单独费用。 发行人可以选择不创建通证名称，并可以使用注册地址访问或搜索资产。 这是一个本地名称，在 Sigchain 之外无效。

如果通证需要一个唯一的全局名称（代码），那么发行人需要创建一个全局名称并将其链接到通证。

SUPPLY 供应量

这是需要生成的同質化通证的总数。 费用随着供应量的增加而增加。 查看费用 here.

通证生成后，通证的最大供应量不能增加。

DECIMAL 小数位数

这是通证的小数位数，增加小数位数也会增加费用。 微交易需要小数单位，并根据通证使用和通证经济学进行设置。

Token Accounts 通证账户:

要创建和使用通证，用户需要了解不同的通证账户和通证属性：

通证创建后，通证生成地址将列在 “Tokens" 通证页面中。 这是通证生成地址，而不是通证账户。 该生成地址归发行通证的用户账户所有，用户需要创建一个通证地址来接收通证。

生成地址中的通证可以被销毁以将它们永久地从流通中移除。 此选项目前在界面中不可用。

 

Token Generation Address 通证生成地址

这是生成特定通证的注册地址，该地址中的通证已停止流通。 发送到通证账户的通证开始流通。 用户可以将通证发送回生成地址以将其从流通中移除。

 

Token account 通证账户:

这是一个普通的通证账户，可以从生成地址或另一个通证账户接收特定的通证。 每个通证账户只被送到一个特定的通证，不能接收任何其他通证。 每个用户都必须创建一个通证账户来交易特定的通证。

通证账户只能接收它们所链接的特定通证。 他们不能接受 NXS 麟社币或任何其他通证。 标准的 “default" 默认、“trust" 或任何 NXS 麟社币账户都不能接受任何通证。

创建后，通证具有三个重要属性：

  • maxsupply - 将存在的最大通证数量
  • currentsupply - 分配给通证账户的流通通证数量
  • balance - 尚未分发的通证数量 (maxsupply - currentsupply)

 

Token Minting Fees 通证铸造费

费用基于您定义的通证可分割单位的数量（供应量和小数的组合）。 最低费用为 1 NXS，此后按线性计算，因此每增加一个有效数字需要额外支付 100 NXS 麟社币。 例如。

100 units = 1 NXS

1000 units = 100 NXS

1000000 units = 400 NXS

10000000000 units = 800 NXS

Create a Token 创建通证

要使用界面创建通证，请按照以下步骤操作：

  • 打开 Nexus 麟社界面，确保钱包已完全同步并登录用户账户（Sigchain）签名链。
  • 在 “Overview" 概览页面的底部，单击 “Users" 用户模块。 这将打开用户页面。
  • 在 “Users" 用户页面的左侧，单击 “Tokens" 令牌选项卡。
  • 在 “Assets“ 资产页面中，单击 “Create a new token” 创建新通证。 这将打开新通证页面。
  • In this page the user has to provide three parameters which define the token. The parameters and their details are given below:
  • Once the parameters are provided, click on the "Create token" button on the bottom of the page.
  • Once the token is confirmed on the blockchain, it will be listed in the "Tokens" page.
  • To check the details of any token, click on the token address in the "Tokens" page and the token details page will open with the details.
