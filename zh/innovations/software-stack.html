软件堆栈

Nexus 麟社架构设计为七层软件堆栈，其中包括一个基于 64 位寄存器的处理虚拟机。 每一层都被指定为相互独立地执行一个专门的处理，为现有的 互联网堆栈（OSI 模型）提供附加功能。 这使麟社成为模块化区块链。

让我们分别深入研究每一层的工作原理：

界面

界面是用户空间，即用于与 dapp 交互的按钮。这一层可以伴随从 AR/VR 到移动界面的各种技术。我们的官方接口层为开发人员提供了一个平台来构建直接嵌入到 Nexus 钱包中的模块和应用程序。

 

逻辑的

这是开发人员应用程序空间的第一层，它形成了大多数应用程序的逻辑。该层直接与其下方的 API 交互，并提供不依赖于直接访问区块链的功能。这方面的一个例子可能是简单的请求，例如：“如果另一个用户完成了此事件，则向该用户发送消息。”我们假设未来这一层也将由人工智能系统驱动。

 

API

该层提供了一个接口，允许开发人员直接访问区块链功能。它基于简单的动词和名词语义，并接受多种编码。它是进入区块链的网关，无需直接访问较低级别的软件堆栈即可使用，使在 Nexus 上的开发就像开发 Web 应用程序一样简单。

 

运营

操作是给寄存器上下文的指令或动作，并定义更复杂的合约逻辑。合约是一个对象，包含：一个寄存器预状态（正在操作的寄存器，从寄存器层向上传递），一个原始操作（每个合约只有一个原始操作）和一组条件（任何数量条件操作）。

 

原始操作：

操作层的当前迭代包含 16 个原始操作和 54 个条件操作和类型。原始操作可以最好地描述为发生在寄存器上的动作，例如：写入、借记、转移、追加等。这些动作本身会导致寄存器以某种方式或形式改变其状态，包括其从一个签名的移动链到另一个。

 

有条件的合同：

有条件的合同是参与方之间的协议，概述了完成交易必须满足的一组要求。它们是允许用户彼此互动的基石，例如合同到期或物品交换。更高级的非托管托管或仲裁形式也是可能的。条件语句对其复杂性没有限制，能够处理一起评估为真或假的条件组。在条件返回 true 的情况下，这允许交易的接收者索取他们的资金或对象（取决于这是转移还是借记）。在收款方无法满足条件的情况下，在发送方设定的时间后，交易将可赎回。

 

登记

寄存器是计算机 CPU（中央处理单元）内最快的存储介质，处理器使用它来缓存计算之间的数据。作为对称为“堆栈”的更古老架构的改进，这已成为计算机设计更理想的架构。

 

Nexus Registers 是一种数据存储系统，可维护不可变的记录和历史记录，包括当前和以前的状态，因此它们可用于记录应用程序的状态。

 

寄存器的所有权可以在 Sigchains 之间转移，因此可以作为资产、代币或简单的对象，例如加密猫。登记册也可以由代币拥有，从而创建它所代表的资产的部分所有权。寄存器层由三个主要组件组成：对象寄存器、状态寄存器和用于条件执行的 64 位基于寄存器的虚拟机。

 

Nexus 区块链上的每个寄存器都有两个主要属性：一个地址（称为寄存器地址），用于确定寄存器数据库中数据的位置，以及一个所有者，它是签名链的创世哈希，目前拥有登记册。

 

对象寄存器：

对象寄存器是用预定结构记录的可编程类型安全对象，该结构描述资产、代币或程序员的对象。 它们可用于多种用途，包括公司股票、数字资产、证书、数据文件和账户。 对象内的数据字段带有定义字段是否可变的说明符，使对象本身能够强制某些字段具有只读或写访问权限。
