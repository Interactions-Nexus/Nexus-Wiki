3次元链（未来）

三维链（3DC）旨在解决“区块链三难困境”。它将把账本转变为一个多层处理系统，以便安全地扩展并具有高度的去中心化。区块链三难困境是一种观点，即安全性、去中心化和可扩展性这三种品质中只有两种是可以同时实现的。

 

3DC 将密码基元链接在一起成为三维不可变对象（3D 块），并具有三个核心维度：信誉通道 (X)、不变性或真实性 (Y) 和时间 (Z)。该架构通过 TAO 框架进行部署。

 

可扩展性

传统区块链的架构类似于在单车道高速公路上驾驶汽车——随着汽车数量的增加，交通就会出现。 Nexus 将“可扩展性”视为一项要求，而不是一项功能。因此，我们设计的协议可以随着更多节点加入网络而扩展，即使资源需求增加也能不受阻碍地进行处理。

 

使用“签名链”、“聚合”和“计算分片”，3DC 创建交易处理的并行通道以生成 L1 层，即 3DC 的基础层。然后使用我们称之为“数据分片”的方法将数据存储在许多节点之间，这样就无需同步和存储整个区块链。 LISP（位置标识符分离协议）和“LLL”（低级库）共同为此形成了通用接口，随着更多节点加入网络，分布式数据存储增加，提供了长期扩展潜力。

 

签名链

Nexus 交易不再使用 UTXO（Unspent Tx Outputs）架构，在这种架构中，您将一笔交易的输出作为另一笔交易的输入，从而导致即使是小交易也需要进行大量昂贵的签名验证。尽管 UTXO 是比特币架构的重要基石，但它已被证明已经过时并且容易受到许多不同类型的攻击和扩展限制。

 

为什么要避免这一点很重要？莱特币 50% 的 UTXO 是无法花费的。

 

作为对传统区块链架构的一种转变，Nexus 设计并实施了一种名为签名链的架构，它充当个人用户级区块链，将您的所有数据包含在一个独特的链中。这种架构提供了更高的扩展特性，因为每笔交易只需要验证一个签名。相反，单个 UTXO 交易可能包含 1000 个输入（因此需要 1000 个签名验证），以便交易少量硬币（少于 0.00001）。此外，签名链不需要 wallet.dat 文件，因为它们可以通过登录凭证（用户名、密码和 PIN）访问。这可以在不牺牲隐私的情况下利用网络上的声誉验证个人的真实性和身份。

下层图书馆 (LLL)

Lower Level Library (LLL) 是 TAO 框架的基础，能够以高可靠性、性能和可扩展性支持其他 Web 技术。 LLL 是一个高性能的“模板库”，旨在为新兴网络技术提供支持。经过数十年的发展，当今的大多数网络技术都笨拙、集中且过度设计。 LLL 简单、强大、轻量级；考虑到模块化。

 

它包含三个主要组件：密码学 (LLC)、数据库 (LLD) 和协议 (LLP)。 Cryptography 组件包含 Quantum Resistant 密码学，Database 组件的性能比 Google 的 LevelDB 高出几个数量级，而 Protocol 组件每秒处理超过 450k 个请求。

 

Lower Level Database (LLD) 是 Nexus 的快速模块化存储引擎，据我们所知，它能够胜过大多数现有的嵌入式数据库引擎。对于 100k 次磁盘写入和读取（一个然后另一个），我们的平均结果约为 0.33 秒。这可以与其他存储引擎相媲美，例如 Google 的 LevelDB。

 

低层协议 (LLP) 是网络层的基本组件，它是一种轻便快速的协议，允许开发人员自定义其数据包设计和消息解释。它通过简单性获得可扩展性，并且能够管理大量并发连接。

 

Lower Level Cryptography (LLC) 是一个轻便高效的库，包含许多有用的密码函数，例如后量子密码学、AES 和 Argon2。该库提供了一组易于访问的高性能加密函数，以确保最大的扩展潜力。一个例子是我们的 FALCON 基准（在 TAO 中使用）在消费级苹果笔记本电脑上验证了 150k 签名，其中 ECDSA（椭圆曲线数字签名算法；用于比特币、以太坊等）仅执行 4k 签名。

 

安全

Nexus 采用多个共识系统相互“制衡”。多样性加强了人类的基因库，同样，它对于去中心化系统的安全性也是同样重要的属性。 3DC 被设计为多层事务处理或“共识”，每一层都聚合来自下一层的数据。在 L2 上执行工作的节点解决 L1 分片中的任何冲突，使用“权益”和“信任”作为“权重”来确定共识。如果发生冲突，则通过数据的有效性来解决，其定义为（信任+权重）。 L3 层将合并来自 L2 的哈希以创建最终的 3D 块。

 

Nexus 非常重视密码学的使用，因为这些功能中的缺陷可能会危及整个网络的安全。我们只采用经过良好测试和彻底同行评审的密码学来支持更高水平的抗量子性。

 

信任与权重

信任被定义为特定用户（签名链）为网络做出贡献的总时间。这个时间是通过节点验证交易数据的一致性和可用性来衡量的。

 

权重定义为给定节点为一次性交易过程提供的实时资源贡献。这可以通过工作量证明 (PoW) 或其他资源（例如会产生提供成本的“股份”）以计算周期来衡量。

 

pBFT + 信誉通道（L1）

当网络接收到交易时，节点会立即开始验证它们。 L1 通道的交易速度将根据商家希望承担的风险而有所不同，从亚秒级速度到五秒不等。对于更高价值的交易，建议他们从下一个共识层的验证中获得额外的权重：L2，将交易速度降低到 15 秒以上。

 

pBFT + PoS 信任网络（L2）

作为现有权益证明系统的扩展，L2 将形成 L1 之上的第二层共识。 L2 层确保安全性和活性，跨分片通信，并从 L1 层解决冲突。它代表了 L1 通道的水平链接，是迈向真正去中心化和可扩展分类账的重要一步。

 

检查和平衡

为了获得最高程度的安全性，决策不能集中在一种形式，因为这会产生“强制”的能力。如果只有一种形式的成本可以提供安全性，那么由于有限的“检查和平衡”，系统很容易被控制。比特币是遭受资源支配或“集中化”的受害者的典型例子。

 

四个组织控制着超过 51% 的比特币算力，这意味着比特币的整个安全性都依赖于它们以及它们做出的决定。这种情况是资源支配导致中心化的一个例子，这导致了提出的解决方案，例如 UASF（用户激活软分叉）和多个比特币分叉，例如比特币现金、比特币 SV、比特币黄金等。

权力下放

由于 PoW 的能量需求大，许多协议已经远离 PoW。极度的竞争性也导致越来越多的资源用于寻找区块，因为传统的 PoW 模型只奖励每个区块的获胜矿工，激励矿工集中资源。

 

委托权益证明 (DPoS) 是 EOS、TRON、LISK 和其他主要用于实现扩容增强的共识机制，它依赖于有限数量的选定区块生产者，产生低程度的去中心化。已经提出了几种用于扩展区块链的解决方案：例如比特币的隔离见证与闪电网络相结合，以及以太坊的 Plasma。尽管很有前途，但两者本质上都依赖于链下解决方案来提供扩展（一种更集中的方法）。他们创建支付渠道或“侧链”，依靠一小群验证者重新提交更新后的余额。更年轻的协议提出了多层系统，尽管我们不知道有任何设计像 3DC 一样强调去中心化。

 

3DC 旨在通过多种方法实现权力下放，包括： L1 信誉通道、L2 和 L3 层上的去中心化池、信誉激励结构和同行发现。

 

L1 信誉通道

与 L2 和 L3 层相比，L1 信誉通道旨在需要少量资源。这是为了能够使用更小的移动设备，从而提供更高级别的去中心化。这是可能的，因为上面的 L2 共识层增加了权重以确保下面通道的安全。声誉是 3DC 在实现高度去中心化的同时维护安全的最后一个要素。它通过 3DC 的所有三层进行聚合，以量化 3D 块的“有效性”。

 

去中心化权益池（L2）

L2 层是 3DC 的核心，负责管理数据聚合和合约处理。这一层也从上面的L3层的矿工那里获得份额，以积累他们的工作并集体奖励矿工。 L3 层包含的份额越多，给定 3D 块的累积权重和信任就越大。因此，3DC 激励 L2 验证者包含尽可能多的份额，以确保他们的 3D 块被接受为 3D 链中最有效的。

 

L2 层由“权益证明”权重驱动，它将共识过程中的所有节点标识为贡献者，因此与现有的权益证明 (PoS) 协议相比，产生更高程度的去中心化。与当前的 PoS 协议相比，3DC 将需要更低的最低余额才能进行质押。

 

去中心化矿池（L3）

该层将使用根据 L2 节点执行的工作计算的基于 PoW 的挖矿份额。共识将由最大的股份价值 + 信任度来决定，以最终就最有效的 3D 区块达成一致。

 

矿工不再有权通过找到获胜的哈希值来确定下一个区块，而是挖矿将成为形成 3DC L3 层的全组活动。向网络提交哈希的矿工执行锁定 L2 交叉链接的工作。这为更加去中心化的共识过程提供了基础设施，同时也继承了挖矿提供的积极特性。

 

同行发现

区块链通常依靠节点直接连接（点对点）的能力来维持分散且均匀分布的拓扑结构。通过能够接受连接请求，节点必须能够被其对等方发现。这很少能实现，并导致比特币只有区区 10% 的可发现节点。

 

或者，Nexus 使用 LISP 覆盖来遍历“NAT”（网络地址转换器）以保持更高程度的节点可用性。 LISP 使用静态端点标识符 (EID)，甚至可以在不同网络（WiFi、手机信号塔等）之间漫游时到达。这大大提高了节点的移动性，允许它们位于互联网上的任何位置、住宅环境中的 NAT 后面、云提供商中以及移动运营商后面，同时仍然可以被发现。

 

声誉激励结构

为了创建健康的全球网络，声誉是去中心化系统运行的重要要求。我们将在 3DC 的所有三个层上实施声誉，作为权重的次要组件，提高整体 pBFT。同样重要的是，声誉可以通过对赢得更高声誉的节点的可变奖励促进激励结构来改善去中心化。系统的长期贡献者可以获得更高的声誉，因此他们的贡献也将获得更高的回报，从而产生了对 Nexus 的长期看法：

 

不是每个人都有钱，但每个人都有时间
