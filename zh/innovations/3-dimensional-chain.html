三维链（未来）

三维链（3DC）旨在解决“区块链三难困境”。它将把分类账本转变为一个多层处理系统，以便安全地扩展并具有高度的去中心化。区块链三难困境是一种观点，即安全性、去中心化和可扩展性这三种品质中只有两种是可以同时实现的。

三维链将密码基元链接在一起成为三维不可逆变物件（3D 区块），并具有三个核心维度：声誉通道 (X)、不变逆性或真实性 (Y) 和时间 (Z)。该架构通过 TAO 麟道框架进行部署。

 

可扩展性

传统区块链的架构类似于在单车道高速公路上驾驶汽车 - 随着汽车数量的增加，交通就会出现。 Nexus 麟社将 “Scalability 可扩展性” 视为一项要求，而不是一项功能。因此，我们设计的协议可以随着更多节点的加入网络而扩展，即使资源需求增加也能不受阻碍地进行处理。

使用 Sigchains 签名链、“聚合” 和 “计算性分片”，3DC 三维链创建交易处理的并行通道以生成 L1 层，即 3DC 三维链的基础层。然后使用我们称之为 “数据分片” 的方法将数据存储在许多节点之间，这样就无需同步和存储整个区块链。 LISP（位置标识符分离协议) 和 LLL (低层库) 共同为此形成了通用接口，随着更多节点加入网络，分布式数据存储增加，提供了长期扩展潜力。

 

签名链

Nexus 麟社交易不再使用 UTXO（未花费交易输出）架构，在这种架构中，您将一笔交易的输出作为另一笔交易的输入，从而导致即使是小交易也需要进行大量昂贵的签名验证。尽管 UTXO 是比特币架构的重要基石，但它已被证明已经过时并且容易受到许多不同类型的攻击和扩展限制。

为什么要避免这一点很重要？50% of Litecoin’s UTXO is unspendable. 莱特币 50% 的 UTXO 是无法花费的。

作为对传统区块链架构的一种转变，Nexus 麟社设计并实施了一种名为签名链的架构，它充当个人用户层区块链，将您的所有数据包含在一个独特的链中。这种架构提供了更高的扩展特性，因为每笔交易只需要验证一个签名。相反，单个 UTXO 交易可能包含 1000 个输入（因此需要 1000 个签名验证），就算是交易少量硬币（少于 0.00001）。此外，签名链不需要 wallet.dat 文件，因为它们可以通过登录凭证（username 用户名称、password 登录密码，和 PIN 操作密码）访问。这可以在不牺牲隐私的情况下利用网络上的声誉验证个人的真实性和身份。

 

聚合

传统区块链中的交易不仅在区块中被引用，它们还与区块一起传输。 虽然这确实包含一些处理的积极特征，但它限制了规模，因为交易需要传输两次，一次是在创建时，一次是在区块本身被广播时。 为了解决这种低效率问题，Tritium 麟氚协议将交易物件与块物件分开存储，并引用区块内的 txid。 这是 “Aggregation 聚合” 的第一种形式，这意味着单个引用可以代表整个交易，从而减少以区块为单位传输的数据。 通过降低网络上成功进行 Finney 芬尼攻击的可能性，这会提高扩展级别并提高安全性。

什么是 Finney 芬尼攻击？ Hal Finney Discovered it in 2011.

 

计算性分片

Computational Sharding 计算性分片对于特定类型节点之间的工作分工是必要的，以创建并行处理数据的 “线道”，类似于高速公路的多车道。尽管计算性分片功能强大，但如果实施不当，它的安全性可能会降低。原因是 “分片” 比整个网络更容易支配，因为它更小。解决这个问题的方法是通过使用 3DC 三维链固有的多层分类帐本（在安全性中有解释）。共识层允许下面的分片比上面的更小，保护冲突免受攻击。

 

数据分片

Data Sharding 数据分片是将要存储在许多节点之间的数据划分。这可以认为是有很多仓库来存放处理和运输后的包裹（数据）。由于每个物件都可以通过其索引哈希 “验证”，因此 3DC 三维链可以提供对远程节点的有限信任的数据分片。

其难处在于，这么多物件和分片的状态如何管理？ LISP 位置指示符分离协议的使用解决了这个问题。 3DC 三维链执行数据分片的方法创建了一个无处不在的 “网络”，其中没有 “IP 互联网协议” 地址，而是 “哈希”。这可以比作在您的网络浏览器中输入一个 txid，然后接收该交易。以这种方式使用 LISP 位置指示符分离协议，我们将使浏览器（或网络术语中的 LLP）打开一个到哈希的连接，该哈希指向保存特定数据的节点组。

这样做的最终结果是，用户可以登录到他们以前从未与网络通信过的节点，使用他们的凭据生成一个 “genesis-id 起源身份” 并打开一个到哈希的连接，利用现有的互联网路由到该节点包含此特定数据的。这样做的好处是网络本身不需要在节点之间添加多余的数据同步来了解数据的保存位置。节点使用覆盖层来路由远程节点请求，从而促成 IP 地址作为存在于 Nexus 麟社美妙世界中的数据哈希。

数据分片是 3DC 三维链实现长期可扩展性的一个重要方面。 Amine 麟胺将为节点提供以 “shard mode 分片模式” 运行的机会，即使在网络承受高负载时也能降低它们的磁盘和内存使用率。 Obsidian 麟耀石中的数据分片将扩展到关键的网络功能，促成节点只需要存储整个链的一部分。

 

多播

除了数据分片中描述的密码结构挑战之外，互联网还必须能够有效地进行路由。我们利用所谓的 “IP Multicast 多播”，它使消息的单次广播能够由一个节点发起，而不是每个节点都需要在验证消息时复制该消息。这可以比作公共演讲者向听众广播消息（多播），而不是进行一对一的对话（单播），然后将消息从一个人传给另一个人。您可以想象这不仅会提高可伸缩性，还会提高消息的完整性（因为流言通常无法反映原来的谈话）。无论有多少节点做为系统的一部分，数据包和交易都将在恒定时间内路由。

 

低层库 (LLL)

低层库 (LLL) 是 TAO 麟道框架的基础，能够以高可靠性、性能和可扩展性支持其他网络技术。 LLL 低层库是一个高性能的 “Template Library 模板库”，旨在为新兴网络技术提供支持。经过数十年的发展，当今的大多数网络技术都笨拙、集中且过度设计。 LLL 低层库简单、强大、轻巧；模块化设计理念。

它包含三个主要组件：低层加密 (LLC)、低层数据库 (LLD) 和低层协议 (LLP)。 加密组件包含抗量子密码学，数据库组件的性能比 Google 谷歌的 LevelDB 高出几个数量级，而协议组件每秒处理超过 450k 个请求。

低层数据库 (LLD) 是 Nexus 麟社的快速模块化存储引擎，据我们所知，它能够胜过大多数现有的嵌入式数据库引擎。对于 100k 次磁盘写入和读取（一个然后另一个），我们的平均结果约为 0.33 秒。这可以与其他存储引擎相媲美，例如 Google 谷歌的 LevelDB。

低层协议 (LLP) 是网络层的基本组件，它是一种轻便快速的协议，允许开发人员自定义其数据包设计和消息解释。通过其简易性取得可扩展性，并且能够管理大量并发连接。

低层加密 (LLC) 是一个轻便高效的库，包含许多有用的密码函数，例如后量子密码学、AES 和 Argon2。该库提供了一组易于访问的高性能加密函数，以确保最大的扩展潜力。一个例子是我们的 FALCON 快速傅立叶采样算法基准（在 TAO 麟道中使用）在普通用户级苹果笔记本电脑上验证了 150k 签名，其中 ECDSA（椭圆曲线数字签名算法；用于比特币、以太坊等）仅执行 4k 签名。

 

安全性

Nexus 麟社采用多个共识系统相互 “制衡”。多样性加强了人类的基因库，同样，它对于去中心化系统的安全性也是同样重要的属性。 3DC 三维链被设计为多层交易处理或 “共识”，每一层都聚合来自下一层的数据。在 L2 上执行工作的节点解决 L1 分片中的任何冲突，使用 “权益” 和 “信誉” 作为 “权重” 来确定共识。如果发生冲突，则通过数据的有效性来解决，其定义为（信誉+权重）。 L3 层将合并来自 L2 的哈希以创建最终的 3D 三维区块。

Nexus 麟社非常重视密码学的使用，因为这些功能中的缺陷可能会危及整个网络的安全。我们只采用经过良好测试和彻底同行评审的密码学来支持更高水平的抗量子性。

 

信誉与权重

信誉被定义为特定用户（签名链）为网络做出贡献的总时间。这个时间是通过节点验证交易数据的持续性和可用性来衡量的。

权重定义为给定节点为一次性交易过程提供的实时资源贡献。这可以通过工作量证明 (PoW) 或其他资源，例如会产生提供成本的 “权益” 以计算周期来衡量。

 

pBFT + 声誉通道（L1）

当网络接收到交易时，节点会立即开始验证它们。 L1 通道的交易速度将根据商家希望承担的风险而有所不同，从亚秒级速度到五秒不等。对于更高价值的交易，建议他们从下一个共识层的验证中获得额外的权重：L2，将交易速度降低到 15 秒以上。

 

pBFT + PoS 信誉网络（L2）

作为现有权益证明系统的扩展，L2 将形成 L1 之上的第二层共识。 L2 层确保安全性和活性，跨分片通信，并从 L1 层解决冲突。它代表了 L1 通道的水平链接，是迈向真正去中心化和可扩展分类账的重要一步。

 

检查和平衡

为了获得最高程度的安全性，决策不能集中在一种形式，因为这会产生 “强制” 的能力。如果只有一种形式的成本可以提供安全性，那么由于有限的“检查和平衡”，系统很容易被控制。比特币是遭受资源支配或 “集中化” 的受害者的典型例子。

四个组织控制着超过 51% 的比特币算力，这意味着比特币的整个安全性都依赖于它们以及它们做出的决定。这种情况是资源支配导致中心化的一个例子，这导致了提出的解决方案，例如 UASF (用户激活软分叉) 和多个比特币分叉，例如比特币现金、比特币 SV、比特币黄金等。

尽管很充满希望，但 UASF 无法达到有效的水平，因为所需的矿工同意百分比太高。

 

去中心化

由于 PoW 工作量证明协议的能量需求大，许多协议已经远离 PoW 工作量证明协议。极度的竞争性也导致越来越多的资源用于寻找区块，因为传统的 PoW 工作量证明协议模型只奖励每个区块的获胜矿工，激励矿工集中资源。

委托权益证明 (DPoS) 是 EOS、TRON、LISK 和其他主要用于实现扩容增强的共识机制，它依赖于有限数量的选定区块生产者，产生低程度的去中心化。已经提出了几种用于扩展区块链的解决方案：例如比特币的 Segregated Witness (SegWit) 隔离见证与 Lightning Network 闪电网络相结合，以及以 ETH Plasma 太坊等离子体。尽管很有前途，但两者本质上都依赖于链下解决方案来提供扩展（一种更集中的方法）。他们创建支付渠道或“侧链”，依靠一小群验证者重新提交更新后的余额。更年轻的协议提出了多层系统，我们还没看到有任何设计像麟社 3DC 三维链一样强调去中心化。

麟社 3DC 三维链旨在通过多种方法实现去中心化，包括： L1 声誉通道、L2 和 L3 层上的去中心化池、声誉激励结构和发掘端点。

 

L1 声誉通道

与 L2 和 L3 层相比，L1 声誉通道旨在需要少量资源。这是为了能够使用更小的移动设备，从而提供更高级别的去中心化。这是可能的，因为上面的 L2 共识层增加了权重以确保下面通道的安全。声誉是 3DC 三维链在实现高度去中心化的同时维护安全的最后一个要素。它通过 3DC 三维链的所有三层进行聚合，以量化 3D 三维链区块的 “有效性”。

 

去中心化权益池 (L2) - 去中心化权益证明协议池

L2 层是 3DC 三维链的核心，负责管理数据聚合和合约处理。这一层也从上面的 L3 层的矿工那里获得份额，以积累他们的工作并集体奖励矿工。 L3 层包含的份额越多，给定 3D 三维链区块的累积权重和信誉就越大。因此，3DC 三维链激励 L2 验证者包含尽可能多的份额，以确保他们的 3D 三维链区块被接受为 3D 三维链中最有效的。

L2 层由 “权益证明” 权重驱动，它将共识过程中的所有节点标识为贡献者，因此与现有的权益证明 (PoS) 协议相比，产生更高程度的去中心化。与当前的 PoS 协议相比，3DC 三维链容许更低的余额就能进行质押。

 

去中心化挖矿池 (L3) - 去中心化工作量证明协议池

该层将使用根据 L2 节点执行的工作量证明协议基于 PoW 的挖矿份额。共识将由最大的股份价值 + 信誉数值来决定，以最终就最有效的 3D 三维链区块达成一致。

矿工不再有权通过找到获胜的哈希值来确定下一个区块，而是挖矿将成为形成 3DC L3 层的全组活动。向网络提交哈希的矿工执行锁定 L2 交叉链接的工作。这为更加去中心化的共识过程提供了基础设施，同时也继承了挖矿提供的积极特性。

 

同行发现

区块链通常依靠节点直接连接（点对点）的能力来维持分散且均匀分布的拓扑结构。通过能够接受连接请求，节点必须能够被其对等方发现。这很少能实现，并导致比特币只有区区 10% 的可发现节点。

 

或者，Nexus 使用 LISP 覆盖来遍历“NAT”（网络地址转换器）以保持更高程度的节点可用性。 LISP 使用静态端点标识符 (EID)，甚至可以在不同网络（WiFi、手机信号塔等）之间漫游时到达。这大大提高了节点的移动性，允许它们位于互联网上的任何位置、住宅环境中的 NAT 后面、云提供商中以及移动运营商后面，同时仍然可以被发现。

 

声誉激励结构

为了创建健康的全球网络，声誉是去中心化系统运行的重要要求。我们将在 3DC 的所有三个层上实施声誉，作为权重的次要组件，提高整体 pBFT。同样重要的是，声誉可以通过对赢得更高声誉的节点的可变奖励促进激励结构来改善去中心化。系统的长期贡献者可以获得更高的声誉，因此他们的贡献也将获得更高的回报，从而产生了对 Nexus 的长期看法：
