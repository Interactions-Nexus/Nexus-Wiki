ONE-STACK 麟堆栈 (未来)

ONE 的首字母缩写词代表 One Nexus Execution One Nexus Execution 麟壹执行堆栈 (麟堆栈)，这是一个旨在取代 OSI (开放系统互连) 参考模型的堆栈。它是 Nexus 麟社协议的基本参考模型。 ONE stack 麟堆栈由 7 层组成，其中两层来自当前的 OSI 开放系统互连。

Signal 信号层

信号层是指卫星和地面站之间的通信。信号层负责载波的物理调制。它无线连接到相控射线天线。它使用免许可的 5.8 GHz 频段进行通信，以开放部署地面和轨道基础设施。它将遵循 MCS (调制和编码方案) 直至 MCS-11，使用 1024-QAM。

 

Location 位置层

第二层是定位器层，负责将数据包路由到正确的地理位置。它将有多种类型的定位器，主要类型是地理空间定位器 (GSL)、宽半径定位器 (WRL) 和互联网协议定位器 (IPL)。路由是在拓扑上独立完成的，不需要管理任何状态来获得完整的路由。它与传统互联网协议 IP 互操作，作为另一组定位器。

 

IDENTIFIER 标识符层

标识符层是数据包裹实际寻址的地方。它就像一台拥有永久电话号码的计算机，无论该特定计算机位于哪个网络，任何人都可以联系。标识符是唯一的并且经过密码验证，这意味着标识符不能被欺骗。它扩展到 IPV6 的 128 位元可用数字空间，但它仅将其用作 IP/NP 之间的通用接口。

 

SESSION 会议

会话层是从 OSI 引入的。 UDP 和 TCP 等协议仍将在 NP 中使用。仍将使用 TLS 等会话层安全性。即使在 IP 上运行 LX-OS 时，仍然会有相同的套接字接口，因为您将始终打开 TCP/NP/IP 到它们的 128 位标识符。这意味着用户可以使用 IP 或 NP，就像使用一个大型互联网一样。

 

会话层在设备之间创建称为会话的通信通道。它负责打开会话，确保它们在数据传输时保持打开和功能，并在通信结束时关闭它们。会话层还可以在数据传输期间设置检查点——如果会话中断，设备可以从上一个检查点恢复数据传输。

 

运输

传输层是从 OSI 引入的。传输层接收在会话层传输的数据，并在发送端将其分成“段”。它负责在接收端重新组装段，将其转回会话层可以使用的数据。传输层执行流量控制，以与接收设备的连接速度相匹配的速率发送数据，以及错误控制，检查数据是否被错误接收，如果没有，则再次请求。

 

共识

共识层负责在 IP 和 NP 上创建信任层。它是管理信任根（例如安全证书）的公共参考点。它由 Nexus 软件堆栈的三层组成：分类帐、注册和操作。这是我们获得 One Nexus Execution 的地方。

 

应用

应用层是人机交互层，应用程序可以在其中访问网络服务，例如网络浏览器和电子邮件客户端。它提供允许软件发送和接收信息并向用户呈现有意义数据的协议。应用层协议的几个示例是超文本传输​​协议 (HTTP)、文件传输协议 (FTP)、邮局协议 (POP)、简单邮件传输协议 (SMTP) 和域名系统 (DNS)。
