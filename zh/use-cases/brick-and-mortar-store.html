🏪 实体店

Nexus 麟社专为现实世界、普通人的日常使用而设计，可以在此用例中理解。 这可以由任何实体店和汽车商店轻松实施，以通过集成 Nexus 麟社来管理商店。

Nexus 麟社可用于支付、发票、物流和跟踪、在线市场。 为了简单起见，我们需要了解如何实现这些东西。

根据位置和当地法规，加密支付可能不可行。 该系统可能需要根据当地法规进行微调。

如何实施

实体店将使用混合网络功能，他们可以在其中接受客户的付款，并使用发票在公共链上接受付款并向经销商付款。 虽然他们可以管理库存并负责私有链上的物流。 这很容易，因为私有链不涉及任何费用，而且数据是私有的。

该商店甚至可以有一个由 P2P API 点对点应用程式界面支持的在线市场，这是一个链上订单簿交换。

商店需要将 Nexus API 麟社应用程式界面集成到全新的销售点 (POS) 零售系统中或集成到现有系统中。

商店的间接费用

店主将不得不运行节点来维护混合网络。 节点数将取决于数据库所需的冗余度。 与现有系统相比，开销非常低。 如果店主需要 3 倍冗余，他需要运行 3 个节点，这些节点将托管在 VPS 或自有硬件或两者的混合上。

一个新的 POS 零售系统或集成到系统中可能会产生额外的成本，但如果我们可以制作一个开源代码，任何商店都可以使用它，从而节省特定的开销。

商店里什么可以实施

实体店和汽车商店可以使用下面列出的一个或所有选项：

付款：

Nexus 麟社界面将有一个销售点支付模块。 这将使客户使用手机钱包付款变得非常容易。 这类似于 Apple pay 或 Google pay，用户扫描二维码发送付款。 该商店甚至可以将 Nexus API 麟社应用程式界面集成到他们现有的系统中。 使用 Query DSL 可以很容易地跟踪所有付款并生成报告。

发票：

商店可以开具发票或接受经销商的发票，这些发票将跟踪所有已完成的付款。 这可以使用接口发票模块或将发票 API 应用程式界面集成到现有系统中来访问。 报告内置于发票 API 应用程式界面中，可以使用由 Query DSL 提供支持的链上搜索进一步缩小范围。

库存跟踪：

Nexus 麟社有一个 Supply API 供应应用程式界面，可用于跟踪库存。 这些数据将存储在混合网络的私有端，该网络不收费、速度非常快且免费。 每件物品在作为物品铸造时都会生成自己的二维码，并且现有的产品条形码可以作为唯一标识符或名称集成到供应物品中。

网上商城：

实体店和汽车商店也可以在即将推出的 P2P 点对点市场上拥有在线市场，可用于交换通证和资产。 店主需要在公链上创建自己的实体商品作为资产，并附上位置和店名进行销售。 用户可以使用该位置搜索他们所在地区可用的资产，他们可以在线购买，店主可以使用公共 Supply API 链供应应用程式界面来跟踪快递发送的货物。
